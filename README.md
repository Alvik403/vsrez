# Консолидация Excel-файлов в шаблон

Веб-приложение на `FastAPI` для загрузки:

- встроенного шаблона `xlsx/xlsm` из проекта или загруженного вручную шаблона
- нескольких исходных `xlsx/xlsm`
- необязательного JSON-конфига с маппингом листов и колонок

Результат: скачивание итогового Excel-файла с заполненным шаблоном.

## Стек

- Python
- FastAPI
- openpyxl
- HTML / JS

## Запуск локально

```bash
python -m venv .venv
.venv\Scripts\activate
pip install -r requirements.txt
uvicorn app.main:app --reload
```

Открыть: [http://127.0.0.1:8000](http://127.0.0.1:8000)

## Запуск в Docker Compose

```bash
docker compose up --build
```

Открыть: [http://127.0.0.1:8000](http://127.0.0.1:8000)

## Что уже реализовано

- встроенный шаблон через `default_template_path`
- опциональная ручная загрузка шаблона `.xlsx/.xlsm`
- загрузка нескольких исходных файлов `.xlsx/.xlsm`
- чтение JSON-конфига из `config/app_config.json` или из загруженного файла
- заполнение листа `Справка по резервам `
- добавление строк в существующие блоки `ЦФО` и `Уровень резерва`
- создание новых блоков `ЦФО` с копированием оформления и новой заливкой
- пересборка зависимого листа `Справка`
- агрегация значений по всем исходным файлам
- сохранение результата в формате шаблона, включая `.xlsm`

## Текущая логика консолидации

Сейчас реализован сценарий под шаблон ВСР:

1. Источник читается с листа `Лист1`.
2. Данные пишутся в лист `Справка по резервам `.
3. Для существующих `ЦФО` и `Уровней резерва` строки дописываются в соответствующий блок.
4. Для нового `ЦФО` создаётся новый блок с тем же оформлением и другим цветом.
5. Зависимый лист `Справка` пересобирается заново по строкам деталей.
6. Остальные листы продолжают работать от формул шаблона.

## Формат конфига

```json
{
  "default_template_path": "Файлы/Шаблон ВСР.xlsm",
  "source_cols_preset": {
    "default": {
      "project": { "header": "Проект" },
      "cfo": { "header": "ЦФО" },
      "date": { "header": "Дата вскрытия" },
      "amount": { "header": "Сумма вскрытия" },
      "reason": { "header": "Причина" },
      "type": { "header": "Тип вскрытия" },
      "level": { "header": "Уровень резерва" },
      "comment": { "header": " Комментарий экономиста" }
    }
  },
  "pages": [
    {
      "name": "Справка по резервам",
      "mode": "reserves_sheet_v1",
      "template_sheet": "Справка по резервам ",
      "helper_sheet": "Справка",
      "helper_start_row": 301,
      "template_start_row": 7,
      "source_sheet": "Лист1",
      "source_preset": "default",
      "source_header_row": 2,
      "source_start_row": 3
    }
  ]
}
```

Описание полей:

- `pages` - правила обработки по листам
- `default_template_path` - путь до встроенного шаблона
- `source_cols_preset` - наборы колонок для исходных файлов
- `mode` - режим заполнения шаблона
- `header` / `header_aliases` - поиск колонки по названию
- `column` - альтернативно можно указать буквенную колонку, например `B`
