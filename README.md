# Консолидация ВСР в Excel-шаблон

Проект заполняет шаблон ВСР на основе одного или нескольких исходных Excel-файлов и возвращает готовый `xlsx/xlsm` без ручной правки в Excel.

Сервис нужен для сценария, где:

- есть эталонный шаблон `ВСР`
- есть исходные файлы со вскрытиями резервов
- нужно дописать новые данные в нужные блоки
- важно сохранить структуру листов, группировку строк, merge-ячейки, стили и макросы шаблона

## Что делает проект

На вход сервис принимает:

- шаблон `xlsx` или `xlsm`
- один или несколько исходных Excel-файлов
- JSON-конфиг с правилами поиска колонок и параметрами листов

На выходе сервис формирует итоговый файл в формате шаблона:

- если шаблон был `xlsm`, результат тоже будет `xlsm`
- макросы сохраняются через `openpyxl` с `keep_vba=True`
- по каждому обработанному листу возвращается краткий отчёт в заголовке `X-Consolidation-Report`

## Какие листы меняются

В текущем режиме `reserves_sheet_v1` сервис работает в первую очередь со следующими листами:

- `Справка по резервам `: основная пересборка блоков `ЦФО -> Уровень резерва -> детали`
- `Справка`: пересборка строк-помощников, связанных с деталями из листа резервов
- `Виды вскрытия 2`: агрегирование по видам вскрытия, комментариям и причинам
- `Справка по ЦФО`: агрегирование по ЦФО, комментариям и причинам

Остальные листы обычно не переписываются напрямую и продолжают работать за счёт формул шаблона.

## Как устроена логика

1. Загружается шаблон и исходные книги.
2. Из исходного файла читаются записи со столбцами `Проект`, `ЦФО`, `Дата вскрытия`, `Сумма вскрытия`, `Причина`, `Тип вскрытия`, `Уровень резерва`, `Комментарий экономиста`.
3. На листе `Справка по резервам ` разбирается уже существующая структура шаблона.
4. Новые записи либо добавляются в существующие блоки, либо создают новые блоки `ЦФО`.
5. После этого пересобираются зависимые листы `Справка`, `Виды вскрытия 2` и `Справка по ЦФО`.
6. В книгу выставляется принудительный пересчёт формул при открытии в Excel.

## Что особенно важно по оформлению

Проект старается сохранять не только значения, но и визуальную структуру шаблона:

- стили ячеек
- высоты строк
- merge-ячейки внутри строки
- `outlineLevel`
- `hidden`
- `collapsed`

Для уже существующих строк используются их собственные шаблоны оформления. Для новых строк копируется оформление соответствующего типа строки из шаблона.

## Структура проекта

- `app/main.py` - HTTP API и загрузка файлов
- `app/excel_service.py` - вся логика чтения и записи Excel
- `app/config.py` - загрузка конфига
- `config/app_config.json` - конфиг по умолчанию
- `templates/index.html` - простая веб-форма
- `tests/test_excel_service.py` - автоматические тесты ключевой логики

## API

### `GET /`

Открывает HTML-страницу для загрузки файлов.

### `GET /api/config/default`

Возвращает текущий конфиг по умолчанию.

### `POST /api/consolidate`

Принимает multipart-форму:

- `template_file` - необязательный шаблон `xlsx/xlsm`
- `source_files` - один или несколько исходных файлов
- `config_file` - необязательный JSON-конфиг

Возвращает готовый Excel-файл для скачивания.

### `POST /api/consolidate/test`

Тестовый маршрут без загрузки файлов. Использует:

- шаблон `Файлы/Шаблон ВСР.xlsm`
- исходник `Файлы/РД С24 от 06.03.2026.xlsx`

Подходит для быстрой локальной проверки кнопкой `Тест` в интерфейсе.

Важно:

- маршрут доступен только когда рядом с проектом есть локальная папка `Файлы`
- в чистой серверной сборке без локальных Excel-файлов кнопка `Тест` не показывается

## Запуск локально

```bash
python -m venv .venv
.venv\Scripts\activate
pip install -r requirements.txt
uvicorn app.main:app --reload
```

После запуска открыть [http://127.0.0.1:8000](http://127.0.0.1:8000).

## Запуск через Docker

```bash
docker compose up --build
```

После запуска сервис будет доступен на [http://127.0.0.1:8000](http://127.0.0.1:8000).

Для сервера после деплоя используйте:

- UI: `http://<server-host>:8000/`
- основной endpoint: `POST http://<server-host>:8000/api/consolidate`
- endpoint получения конфига: `GET http://<server-host>:8000/api/config/default`

## Тесты

```bash
pytest -q
```

Сейчас в тестах отдельно проверяется:

- сохранение высот строк и уровней группировки на листе резервов
- сохранение шаблонов строк на листе `Справка`
- сохранение видимости групп на листе `Справка по ЦФО`

## Ограничения

- Проект использует `openpyxl`, поэтому не все возможности Excel сохраняются абсолютно побайтно.
- При открытии Excel может пересчитать формулы заново.
- Условное форматирование, slicer-объекты и некоторые Excel-расширения `openpyxl` читает с ограничениями.

## Краткое описание для бизнеса

Это сервис для автоматической сборки ВСР из исходных файлов резервов. Он снимает ручную работу по копированию строк в шаблон, пересобирает связанные сводные листы и старается сохранить оформление итогового файла таким, как в исходном шаблоне Excel.
